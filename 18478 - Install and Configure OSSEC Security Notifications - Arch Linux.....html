Setting up OSSEC security notifications, Security, Monitoring

Install and Configure OSSEC Security Notifications - Arch Linux

Difficulty: 2

Time: 45 Minutes

This article explains installation and configuration of OSSEC security notifications on Arch Linux

You will need access to sudo privileges to complete all steps.

Prerequisites

  1. For sending mail from a local SMTP server, you will need to install Postfix.
  2. Update the server using following command.
    sudo pacman -Syu
  3. Install following packages.
    sudo pacman -Sy base-devel
    sudo pacman -Sy inotify-tools

Download OSSEC and Verify the package

You will need to download OSSEC tarball and a file with its cryptographic checksum to verify you are installing a valid software. We will generate MD5 and SHA1 checksum and compare with those in checksum file.
  1. Download OSSEC with following command.
    sudo wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1.tar.gz
    And for checksum file use following command.
    sudo wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1-checksum.txt
  2. Verify the checksum with cat command.
    cat ossec-hids-2.8.1-checksum.txt
    You will get:
    MD5(ossec-hids-2.8.1.tar.gz)= c2ffd25180f760e366ab16eeb82ae382
    SHA1(ossec-hids-2.8.1.tar.gz)= 0ecf1df09558dc8bb4b6f65e1fb2ca7a7df9817c
  3. Generate the MD5sum of tarball.
    md5sum ossec-hids-2.8.1.tar.gz
    You will get:
    c2ffd25180f760e366ab16eeb82ae382 ossec-hids-2.8.1.tar.gz
    This should match with the MD5 checksum in file. Same will go for SHA1 as shown
    sha1sum ossec-hids-2.8.1.tar.gz
    It should give output as:
    0ecf1df09558dc8bb4b6f65e1fb2ca7a7df9817c ossec-hids-2.8.1.tar.gz
    It should match with the SHA1 checksum from the file.

Install OSSEC

  1. You can install OSSEC in many mode including server, agent, local or hybrid mode. We will install in local mode in this article. Expand the file with:
    sudo tar -zxf ossec-hids-2.8.1.tar.gz

    If you get an error for tar command type yum install tar to install tar.

    Change the directory to the one we just expanded.
    cd ossec-hids-2.8.1
    Initiate the installation with:
    sudo ./install.sh
  2. It will prompt you to answer a few questions. First question will be to choose a language. You may choose your language as you want. To use default press Enter.

    You will see this:

    				OSSEC HIDS v2.8 Installation Script - http://www.ossec.net
				
				 You are about to start the installation process of the OSSEC HIDS.
				 You must have a C compiler pre-installed in your system.
				 If you have any questions or comments, please send an e-mail
				 to dcid@ossec.net (or daniel.cid@gmail.com).

				  - System: Linux ubuntu 3.13.0-55-generic
				  - User: root
				  - Host: ubuntu


				  -- Press ENTER to continue or Ctrl-C to abort. -
    Press Enter to continue.
  3. Next it will ask you for the kind of installation. So type local and press Enter. You will get:
    Choose where to install the OSSEC HIDS [/var/ossec]:
    Accept default by pressing Enter.
  4. It will ask for email notification.
    Do you want e-mail notification? (y/n) [y]:
  5. Press Enter; it will ask your email address where you want to receive notifications.
    What's your e-mail address?
  6. Type your email address. It will show.
    We found your SMTP server as: advaiya-com.mail.protection.outlook.com.
    - Do you want to use it? (y/n) [y]:
    Press Enter unless you want to use some specific SMTP.

  7. It will ask you about what checks it should perform. You can press Enter to accept default.

    1. Press Enter for integrity check daemon.
      Do you want to run the integrity check daemon? (y/n) [y]:
      - Running syscheck (integrity check daemon).

    2. To detect rootkit, press Enter.

      Do you want to run the rootkit detection engine? (y/n) [y]:
      - Running rootcheck (rootkit detection).

    3. For Active response, press Enter.

      Active response allows you to execute a specific command based on the events received.
      Do you want to enable active response? (y/n) [y]:

    4. Press Enter for default firewall drop response.

      Do you want to enable the firewall-drop response? (y/n) [y]:
      - firewall-drop enabled (local) for levels >= 6

  8. In next prompt it will ask you for more IP addresses but that's not necessary. List files to be monitored will be displayed. You can add more files after installation, so press Enter. It may take a few minutes to install.

    On successful installation you will get 
    					Unknown system. No init script added.

					- Configuration finished properly.

					- To start OSSEC HIDS:
					/var/ossec/bin/ossec-control start

					- To stop OSSEC HIDS:
					/var/ossec/bin/ossec-control stop

					- The configuration can be viewed or modified at /var/ossec/etc/ossec.conf


					Thanks for using the OSSEC HIDS.
					If you have any question, suggestion or if you find any bug,
					contact us at contact@ossec.net or using our public maillist at
					ossec-list@ossec.net
					( http://www.ossec.net/main/support/ ).

					More information can be found at http://www.ossec.net

					---  Press ENTER to finish (maybe more information below). ---
    Now we will start the OSSEC.

Start OSSEC

  1. OSSEC is configured to start at boot but first time you will need to initiate it manually.

    Current status can be checked with

    sudo /var/ossec/bin/ossec-control status
    You will get.
    ossec-monitord not running...
    ossec-logcollector not running...
    ossec-syscheckd not running...
    ossec-analysisd not running...
    ossec-maild not running...
    ossec-execd not running...
  2. To start the OSSEC, type:
    sudo /var/ossec/bin/ossec-control start
    Following will be displayed.
    Starting OSSEC HIDS v2.8 (by Trend Micro Inc.)... Started ossec-maild...
    Started ossec-execd...
    Started ossec-analysisd...
    Started ossec-logcollector...
    Started ossec-syscheckd...
    Started ossec-monitord...
    Completed.
  3. Right after starting the OSSEC, you will get a mail as shown. 
    			OSSEC HIDS Notification.
			2015 Aug 17 02:16:34

			Received From: ubuntu->ossec-monitord
			Rule: 502 fired (level 3) -> "Ossec server started."
			Portion of the log(s):

			ossec: Ossec started.
    You will get such emails whenever something it is configured to monitor happens.

Configure OSSEC

Here you will get an overview of configuration settings and its directory structure.

  1. Directory structure

    Its root directory /var/ossec can only be accessed with root privileges.
    • Its main configuration file resides in /var/ossec/etc.
    • /var/ossec/rules contains predefined rules.
    • /var/ossec/bin contains command used to manage OSSEC.
    • /var/ossec/logs/ossec log contains logs of the errors.

  2. Main configuration file location

    The main configuration file is located at /var/ossec/etc/ossec.conf. Open it in any editor, make sure you have sudo privileges. First configuration option you will see is the email configuration you specified while installing. You can change it, if required at this place.
    <global>
    <email_notification>yes</email_notification>
    <email_to>email@example.com</email_to>
    <smtp_server>mail.example.com.</smtp_server>
    <email_from>ossecm@ossec_server</email_from>
    </global>
    • To change the number of email per hour you can add here the following line.
      <email_maxperhour>N</email_maxperhour>

      N is the number of emails. Make sure your <email_from> address contains a valid domain part as some third-party email provider may drop otherwise.

      If you are not getting emails as expected you can check logs here /var/ossec/logs/ossec.log

  3. Frequency of scan

    The <syscheck> section starts like this.
    <syscheck>
    <!-- Frequency that syscheck is executed - default to every 22 hours -->
    <frequency>79200</frequency>

    So by default system check is run in every 22 hours. You can change it.

    To enable alerts for new file creation, add the below mentioned line
    <alert_new_files>yes</alert_new_files>
    It should read.
    <syscheck>
    <!-- Frequency that syscheck is executed - default to every 22 hours -->
    <frequency>79200</frequency>
    <alert_new_files>yes</alert_new_files>

  4. Directory and file configuration

    Right after that there is a list of system directory that it monitors as shown.
    <!-- Directories to check (perform all possible verifications) -->
    <directories check_all="yes">/etc,/usr/bin,/usr/sbin</directories>
    <directories check_all="yes">/bin,/sbin</directories>
    Add report_changes="yes" realtime="yes" to each line to enable real time monitoring as shown.
    <!-- Directories to check (perform all possible verifications) -->
    <directories report_changes="yes" realtime="yes" check_all="yes">/etc,/usr/bin,/usr/sbin</directories>
    <directories report_changes="yes" realtime="yes" check_all="yes">/bin,/sbin</directories>
    You can add more directories to monitor here. For exmample to add /home/user_name and /var/www/ add following line here.
    <directories report_changes="yes" realtime="yes" restrict=".php|.js|.py|.sh|.html" check_all="yes">/home/user_name,/var/www</directories>
    restrict setting tells only to monitor specified formats.

  5. Local rules location

    Next files you would want to modify reside in /var/ossec/rules. In this directory if you use ls -lgG command, you will get a list of bunch of xml files but only two files local_rules.xml and ossec_rules.xml is of our use. The former file is used to add local rules and latter for default rules definitions. You will need sudo privileges to access these files.

    When a file is added to a monitored directory the rule 554 is fired from ossec_rules.xml. By default it doesn't send alert on firing of this rule. so you'll need to change the settings.

    1. Default rule definitions in ossec_rules.xml are useful to look at so we can modify and copy them into our local files. Rule 554 looks like:
      <rule id="554" level="0">
      <category>ossec</category>
      <decoded_as>syscheck_new_entry</decoded_as>
      <description>File added to the system.</description>
      <group>syscheck,</group>
      </rule>

      When level is set to "0" no alert will be sent. We will raise the alert level.

    2. Copy the rule 554 and open the file /var/ossec/rules/local_rules.xml in any editor and add the new rule at the end within the group tag. Change the level to "7".
      <rule id="554" level="7" overwrite="yes">
      <category>ossec</category>
      <decoded_as>syscheck_new_entry</decoded_as>
      <description>File added to the system.</description>
      <group>syscheck,</group>
      </rule>
    3. Save and close the file. Now restart the OSSEC using:
      /var/ossec/bin/ossec-control restart

      You will get a mail when it is restarted.

  6. Triggering file change alerts

    When you add any file in directory to be monitored by OSSEC or you update or delete a file you should be getting alters. It doesn't send real time notification for adding a file but only for modification or deletion. Alerts of file addition go after full system check which is handled by frequency check time in ossec.conf.

Next Step

After following all these steps you will be able to set up OSSEC notifications. To look for more customization, you can refer to its Official Documentation.
