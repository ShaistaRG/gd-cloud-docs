Install CouchDB From Source - Ubuntu, Debian

Difficulty: 4
Time: 30 minutes

CouchDB database is known to embrace the entire web and belongs to NoSQL family. It is a document-oriented NoSQL database that uses JSON to store data, JavaScript as its query language using MapReduce, and HTTP for an API.

In CouchDB, each database is a collection of independent documents. Each document keeps its own data and self-contained schema. Multiple database may be accessed by an application, may be one stored on mobile and other on server. Revision information in stored in document metadata, which is required to merge changes occurred while the databases were disconnected. This articles will assist you to install CouchDB from source in Ubuntu and Debian.

You need access to sudo privileges to complete these steps.

Install dependencies

Make your machine ready to install CouchDB.

  1. Update the packages.
    $ sudo apt-get udpate
  2. Install tools to compile CouchDB.
    $ sudo apt-get install -y build-essential
  3. To run erlang and other dependencies, use the following command:
    $ sudo apt-get install -y erlang-base erlang-dev erlang-nox erlang-eunit
  4. Install the other required libraries.
    $ sudo apt-get install -y libmozjs185-dev libicu-dev libcurl4-gnutls-dev libtool

Fetch the source

  1. Download the copy of source. Go to the directory where your code will reside.
    $ cd /usr/local/src
  2. Fetch source using following command:
    $ sudo curl -O http://apache.mirrors.tds.net/couchdb/source/1.6.1/apache-couchdb-1.6.1.tar.gz
  3. Extract the files.
    $ sudo tar xvzf apache-couchdb-1.6.1.tar.gz
  4. Change directory to the one we just expanded.
    $ cd apache-couchdb-1.6.1.tar.gz

Configure and install

  • To compile a newly spun CouchDB server, install the source code using following command:
    $ ./configure
    $ sudo make && sudo make install

Finalize the steps

  1. CouchDB runs as user Couchdb, so create a new user with following command:
    $ sudo adduser --disabled-login --disabled-password --no-create-home couchdb
  2. It will ask you for Real name, Room Number etc. you can skip these. Give rights to the new user you created on the CouchDB's files.
    $ sudo chown -R couchdb:couchdb /usr/local/var/log/couchdb /usr/local/var/lib/couchdb /usr/local/var/run/couchdb
  3. Install the CouchDB as service and let it run on boot.
    $ sudo ln -s /usr/local/etc/init.d/couchdb /etc/init.d
    $ sudo update-rc.d couchdb defaults
  4. Run the CouchDB service.
    $ sudo service couchdb start

Conclusion

In this article we reviewed the steps involved with the installation of CouchDB in Ubuntu and Debian. You can see in terminal the CouchDB running as a service.
