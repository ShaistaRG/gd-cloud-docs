Set Up WordPress Domain Mapping - Debian, Ubuntu

Difficulty: 1
Time:5 minutes

The WordPress MU Domain Mapping plugin allows you to point multiple domains to one WordPress account. This means you can map any site to another domain. In our tutorial, we’ll map our subdomains to independent standalone domains. For instance, coolexampleproduct.com would redirect to product.coolexample.com, a subdomain of your primary site, coolexample.com.

Prerequisites

You should have WordPress multisite installed. See Set up WordPress Multisite in NGINX - Debian Ubuntu for instructions:

Download the plugin

  1. Download the plugin:
    cd ~ wget Download the World Press Domain mapping plugin
  2. Install unzip
    sudo yum install unzip
    unzip wordpress-mu-domain-mapping.latest-stable.zip
  3. Then move the files to the WordPress /wp-content/plugins/directory.
    sudo mv wordpress-mu-domain-mapping
    /var/www/html/wp-content/plugins/
  4. Copy the sunrise.php into our WordPress document root.
    sudo cp /var/www/html/wp-content/plugins/wordpress-mu-domain-mapping/sunrise.php
    /var/www/html/wp-content/

Edit your wp-config file

  1. Let’s go to our WordPress document root.
    cd /var/www/html
  2. Open your wp-config file.
    sudo vim wp-config.php
  3. And add this line before /* That's all, stop editing! Happy blogging.
    */
    define('SUNRISE', 'on');
  4. Save and close.
  1. Restart nginx
    sudo service nginx restart
  1. Return to your wp-admin window

  2. In the top navigation bar, you’ll see My Sites. Hover/mouse over it, and from the drop-down menu, select Network Admin, then Plugins
    .

    You’ll see 3 plugins. The WordPress MU Domain Mapping plugin should be the 3rd.

  3. Click Network Activate under this plugin. You’ll see this activate and the plugin box will highlight.
    LH nav bar: Settings ? Domain Mapping
  4. Under Domain Options
  5. Uncheck Remote Login
  6. Check Permanent Redirect
  7. Uncheck Redirect administration pages to site's original domain
  8. Click Save.

Your subdomains will now redirect to the external domains you’ve defined (product.coolexample.com redirects to coolexampleprdouct.com, for example). This includes the administration pages (/wp-admin). In the next step we will be mapping a domain name to each site based on its site ID. There’s more than one way to find the ID of a site but for easier administration we will create a simple WordPress Must-use plugin that displays an additional ID column on the Sites page.

Map a domain name to each site based on site ID

  1. In your terminal, create a directory to house the plugin.
    cd ~
    sudo mkdir /var/www/html/wp-content/mu-plugins
  2. Create a new file.
    sudo vim /var/www/html/wp-content/mu-plugins/wpms_blogid.php

    It’s a new file so it’ll be blank.

  3. Copy this whole code block and paste it into your file.
    <pre>>?php
    add_filter( 'wpmu_blogs_columns', 'do_get_id' );
    add_action( 'manage_sites_custom_column', 'do_add_columns', 10, 2 );
    add_action( 'manage_blogs_custom_column', 'do_add_columns', 10, 2 );
    function do_add_columns( $column_name, $blog_id ) {
    if ( 'blog_id' === $column_name )
    echo $blog_id;
    return $column_name;
    }

    function do_get_id( $columns ) {
    $columns['blog_id'] = 'ID';
    return $columns;
    }</pre>
  4. Save and close.

Back in the WP-Admin browser window: The Sites > All Sites section should now show an additional ID column.

Next Steps

Congratulations! Each site that you’ve defined with its own domain name can now be accessed that way. You can test it by visiting one of those domains. You’ll see the URL redirect to the subdomain of your primary site. Now each site can be maintained separately through its own WordPress admin panel. Just append /wp-admin/ to the URL, such as coolexampleproduct.com/wp-admin/.
