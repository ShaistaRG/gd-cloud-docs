Deploy Rails Apps with Passenger and NGINX - Arch Linux

Difficulty: 2

Time: 30 minutes

Rails enables web developers to rapidly build quality applications, while Nginx is a lightweight, high-performance web server. In this article you'll learn to deploy a Rails application on an Nginx web server using the simple and powerful features of Phusion Passenger. This task assumes that you have sudo privileges.

Prerequisites

'Initial Server Setup', FQDN pointing to server. Merge distros to forks if steps are identical across the fork. You may also refer 'Deploy Rails Apps with Passenger and Apache.

Step 1: Install Ruby on Rails using RVM in Arch Linux

Step 2: Install Nginx and Passenger

  1. Install Passenger and Nginx packages.
    sudo pacman -S nginx
    gem install passenger
  2. Run the dependencies before you install Nginx.
    sudo chmod o+x "/home/demo"
    rvmsudo passenger-install-nginx-module
  3. For language, select Ruby, and then press the Enter key.
    Use <space> to select. If the menu doesn't display correctly, ensure that your terminal supports UTF-8.

Step 3: Configure a test Rails application

  1. Create a directory.
    sudo mkdir -p /var/www
  2. Move to the www directory, and create a Rails application.
    cd /var/www
    rails new HelloWorld
    cd HelloWorld
  3. Create sample resources:
    rails generate scaffold Task title:string note:text
  4. Test your application by running the following command in the application directory:
    ruby -s

Step 4: Configure Nginx

  1. Open the configuration file nginx.conf.
    sudo vim /etc/conf/nginx.conf
  2. Uncomment the following line. 
         passenger_root /usr/lib/ruby/vendor_ruby/phusion_passenger/locations.ini;
     passenger_ruby /home/deploy/.rbenv/shims/ruby; # If you use rbenv
  3. Add the following lines at the end of the nginx.conf file. 
     server { 
     listen 80 default_server; 
     listen [::]:80 default_server ipv6only=on;
        
    server_name mydomain.com;  //your server name 
    passenger_enabled on;
       
    rails_env    production; 
    root      /var/www/HelloWorld/public;//Rails application path

 # redirect server error pages to the static page /50x.html
 error_page   500 502 503 504  /50x.html;
 location = /50x.html 
     {
         
       root   html;
       
     }

    }
  4. Restart the Nginx service.
    sudo systemctl start nginx
