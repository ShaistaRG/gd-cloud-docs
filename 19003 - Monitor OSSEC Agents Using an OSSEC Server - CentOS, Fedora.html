Setting up OSSEC to monitor Agents, Security and Monitoring

Monitor OSSEC Agents using OSSEC Server - CentOS, Fedora

Difficulty: 4
Time: 1 hour

OSSEC (Open Source Host-based Intrusion Detection System) is a security monitoring system.

This article explains how to monitor OSSEC agents with a OSSEC server on CentOS and Fedora.

You will need access to sudo privileges to complete all the steps.

Prerequisites

  • Two separate Ubuntu droplets are required - one for server and other for agent.
  • Iptable should be enabled on both the droplets.

Download OSSEC and verify on the server and the agent

  1. Update the package database.
    sudo yum update
  2. Install following packages on server:
    sudo yum groupinstall 'Development Tools'
    sudo yum install inotify-tools
  3. On the the agent, install the following packages:
    sudo yum groupinstall 'Development Tools'
  4. Install the following package on both the servers, to save any change you make in iptables.
    sudo yum install iptables-services
    sudo systemctl enable iptables
    You can use following command to save iptables rule permanently whenever you make any change or add new rules.
    sudo service iptables save
  5. On both the droplets, download OSSEC and its checksum file.
    wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1.tar.gz
    wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1-checksum.txt
  6. After downloading both the files, type the following command to verify md5sum of the compressed tarball:
    md5sum -c ossec-hids-2.8.1-checksum.txt
  7. The output should be:
    ossec-hids-2.8.1.tar.gz: OK
    md5sum: WARNING: 1 line is improperly formatted
  8. Type the following command to verify SHA1 of the compressed tarball.
    sha1sum -c ossec-hids-2.8.1-checksum.txt
  9. You should get output as:
    ossec-hids-2.8.1.tar.gz: OK
    sha1sum: WARNING: 1 line is improperly formatted
    You can ignore the warnings, as the OK next to tar file confirms that the file is good to go.

Install the OSSEC server

  1. Type the following command to untar the file.
    sudo tar xf ossec-hids-2.8.1.tar.gz
    If you get an error for tar command, type:
    sudo yum install tar
  2. Change directory to the one you just expanded.
    cd ossec-hids-2.8.1/
  3. Start the installation.
    sudo ./install.sh
  4. During the installation you will be prompted to answer a few questions, just press Enter to accept the default answers. In first question you will be asked to choose a language which is by default English(en), press Enter to accept it.
  5. In the next question you will be asked for the type of installation, type server and press Enter.
    1- What kind of installation do you want (server, agent, local, hybrid or help)? server
  6. For the rest of the questions, please accept default. Question on email should be answered with a valid email id, where all the notifications will be sent.
  7. After all the questions the installation will start.On the successful installation, you will see the following: 
    				In order to connect agent and server, you need to add each agent to the server.
				Run the 'manage_agents' to add or remove them:

				/var/ossec/bin/manage_agents

				More information at:
				http://www.ossec.net/en/manual.html#ma

Configure OSSEC Server

  1. OSSEC agent is not yet installed but we can configure the server.To access or modify the OSSEC's files,switch to the root user.
    sudo su
  2. Change directory to the OSSEC's configuration file's directory.
    cd /var/ossec/etc
  3. Open ossec.conf file.
    vim ossec.conf
    Here on the top you can see a few fields. Following is the description of these fields.
    • <email_to> is the email you entered during installation.
    • <email_from> is where the emails appear to be coming from.
    • Change <smtp_server> to localhost if you have your own email server installed on same the host as OSSEC.
    You can update these settings as per your need.After all the changes, the section will look like: 
    				<global>
					<email_notification>yes</email_notification>
					<email_to>email@coolexample.com</email_to>
					<smtp_server>mail.coolexample.com.</smtp_server>
					<email_from>user@coolexample.com</email_from>
				</global>
  4. Now save the file and close it by pressing esc and following characters followed by pressing Enter.
    :wq
  5. Start OSSEC
    /var/ossec/bin/ossec-control start

Install OSSEC Agent

  1. Type the following command to untar the file.
    sudo tar xf ossec-hids-2.8.1.tar.gz
  2. Change into the following directory:
    cd ossec-hids-2.8.1/
  3. Start the installation.
    sudo ./install.sh
  4. Most of the prompts are same as before, except a few. When asked about the type of installation, type agent and press Enter
    1- What kind of installation do you want (server, agent, local, hybrid or help)? agent
  5. When asked about the server IP address, type IP address of your server.
    3.1- What's the IP Address or hostname of the OSSEC HIDS server? : server_ip
  6. For other questions, accept default by pressing Enter. On the successful installation you will see: 
    					You first need to add this agent to the server so they
					can communicate with each other. When you have done so,
					you can run the 'manage_agents' tool to import the
					authentication key from the server.

					/var/ossec/bin/manage_agents

					More information at:
					http://www.ossec.net/en/manual.html#ma
    Server and agent both have been installed but they can't communicate.

Add Agent to the Server

You are already operating as a root from the previous step, so you don't need to use sudo now.

  1. Type following command:
    /var/ossec/bin/manage_agents
  2. You will see options as shown below, type A and press Enter. 
    				(A)dd an agent (A).
				(E)xtract key for an agent (E).
				(L)ist already added agents (L).
				(R)emove an agent (R).
				(Q)uit.
				Choose your action: A,E,L,R or Q: a
  3. You will be asked for the name, IP, and ID of the agent. Keep the name of the agent unique, and then enter the IP of your agent. For the ID you can accept default value by pressing Enter. 
    				- Adding a new agent (use '\q' to return to the main menu).
				Please provide the following:
				* A name for the new agent: agent-name
				* The IP Address of the new agent: ip_of_agent
				* An ID for the new agent[001]:
				Agent information:
				ID:001
				Name:agentUbuntu
				IP Address: ip_of_agent

				Confirm adding it?(y/n): y
				Agent added.
  4. You will be returned to main menu. Press A, Enter, and then give the ID of the agent to fetch the agent key. Make sure to keep a record of the key. 
    				Choose your action: A,E,L,R or Q: e

				Available agents:
				ID: 001, Name: agent-name, IP:ip_of_agent
				Provide the ID of the agent to extract the key (or '\q' to quit): 001

				Agent key information for '001' is:
				MDAxIGFnZW50VWJ1bnyEwNjI5MjI4ODBhMDkzMzA4MR1IXXwNC4yMzYuMjIyLjI1MSBiMTI2U3MTI4YWYzYzg4M2YyNTRlYzM5M2FmNGVhNDYTIwNDE3NDI1NWVkYmQw

				** Press ENTER to return to the main menu.
  5. Press Enter to return to main menu and then Q to quit.

Import Key to Agent from Server

These actions are to be completed on Agent. Type sudo su to change to root to start.
  1. Type following command.
    /var/ossec/bin/manage_agents
  2. You will get the following options: 
    				(I)mport key from the server (I).
				(Q)uit.
				Choose your action: I or Q: i
    After choosing I, follow the direction to paste the key from the server, and then confirm when when asked.
  3. Press Enter to return to the Main Menu, and then choose Q to quit. 
    					Confirm adding it?(y/n): y
					Added.
					** Press ENTER to return to the main menu.

Allow UDP Port 1514 traffic through firewalls

Communication on both sides take place on UDP port 1514, so you will have to add a rule to iptables for both ends.

  1. Temporarily remove the drop rule on both ends.
    sudo iptables -D INPUT -j DROP
    If you don't have a drop rule, it might give you a message of "bad rule".
  2. Add following rule at OSSEC server with agent's IP:
    iptables -A INPUT -p UDP --dport 1514 -s ip_of_agent -j ACCEPT
  3. Now add the following rule on agent's side with server's IP:
    iptables -A INPUT -p UDP --dport 1514 -s ip_of_server -j ACCEPT
  4. These rules should persist after reboot.We have installed iptables-services to take care of this. Type following command to save changes:
    sudo service iptables save

Restart OSSEC Agent and Server

  1. Restart both the servers.
    /var/ossec/bin/ossec-control restart
  2. You can start OSSEC agent from server using the following command:
    /var/ossec/bin/agent_control -R 001
    001 is the agent ID. You should get the following message:
    agent-name-ip_of_agent is active.

Nest Steps

After following all these steps, you should be receiving alerts from both the ends. You may further customize settings for file addition or real time alerts. For configuration settings you can refer Install and Configure OSSEC Security Notifications - CentOS, Fedora

If you want to monitor more than two droplets, add them as OSSEC agents using same steps as explained. If you come across any issue during set up you can go through the logs in /var/ossec/logs/ossec.log file.
