Deploy Rails Apps with Passenger and Nginx - Fedora

Difficulty: 2

Time: 30 minutes

Rails enables web developers to rapidly build quality applications, while NGINX is a lightweight, high-performance web server. In this article you will learn to deploy a Rails application on an NGINX web server using the simple and powerful features of Phusion Passenger. This task assumes that you have sudo privileges.

Prerequisites

To set up your Fedora server, refer to the article: 17322 - Initial Server Setup - Fedora. You need to ensure that the fully qualified domain name (FQDN) points to the server. After you have set up the Fedora server, you will need to 17536 - Install Ruby on Rails using RVM - Fedora. You may also refer to 17719 - Deploy Rails Apps with Passenger and Apache - Fedora link.

Install NGINX and Passenger

  1. Install Passenger and update all packages of operating system..
    gem install passenger
    sudo yum -y update
  2. Install a dependency and provide required permission to home directory.
    sudo yum install libcurl-devel
    sudo chmod o+x "/home/UserName"
  3. Install NGINX passenger.
    rvmsudo passenger-install-nginx-module
  4. For language, select Ruby, and then press the Enter key.
    Use <space> to select. If the menu doesn't display correctly, ensure that your terminal supports UTF-8.
    • Ruby
    • Python
    • Node.js
    • Meteor
  5. Select 1 and press Enter to continue. This action downloads, compiles, and installs the NGINX files with Passenger support.
    1. Yes: download, compile and install Nginx for me.(recommended)
    The easiest way to get started. A stock Nginx 1.4.4 with Passenger support, but with no other additional third party modules, will be installed for you to a directory of your choice.
  6. Save and exit.
    wq!

Set up a test Rails Application

  1. Create a directory
    sudo mkdir -p /var/www
  2. Provide all required Permission and go to that directory.
    sudo chmod -R 0777 /var/www/
    cd /var/www
  3. Create a Rail application and go to application directory.
    rails new HelloWorld
    cd HelloWorld
  4. Create a sample databases and generate a scaffold controller to manage the production database.
    RAILS_ENV=development rake db:migrate
    RAILS_ENV=production rake db:migrate
    rails generate scaffold Task title:string note:text
  5. Test your application by running the following command in the application directory:
    ruby -s

Configure NGINX

  1. Open the following configuration file of NGINX.
    sudo vim /etc/nginx/conf/nginx.conf
  2. Uncomment the below mentioned line right way
    passenger_root /usr/lib/ruby/vendor_ruby/phusion_passenger/locations.ini;
    passenger_ruby /home/deploy/.rbenv/shims/ruby; # If you use rbenv
  3. Add the following line in this file:
    server {
    listen 80 default_server;
    listen [::]:80 default_server ipv6only=on;
    server_name mydomain.com; //your server name
    passenger_enabled on;
    rails_env production;
    root/home/deploy/myapp/current/public; //Rails application path

    # redirect server error pages to the static page /50x.html
    error_page 500 502 503 504 /50x.html;
    location = /50x.html
    {
    root html;
    }
    }
  4. Save and exit.
    wq!
  5. Restart the NGINX service
    sudo service nginx restart
