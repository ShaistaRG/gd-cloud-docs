Set Up a Local OSSEC Installation - Arch Linux

Difficulty: 4
Time: 45 minutes

This article explains the installation and configuration of OSSEC security notifications on Arch Linux

You will need access to sudo privileges to complete all steps.

Prerequisites

  1. For sending mail from a local SMTP server, you will need to install Postfix.
  2. Update the server using following command
    sudo pacman -Syu
  3. Install the packages 
    					sudo pacman -Sy base-devel 
					sudo pacman -Sy inotify-tools

Download OSSEC and Verify the package

You will need to download Ossec tarball and a file with its cryptographic checksum to make sure that you are installing a valid software. Then you can generate MD5 and SHA1 checksum and compare these with the checksum file.
  1. Download ossec with the following command.
    sudo wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1.tar.gz
  2. To download the checksum file, use the following command:
    sudo wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1-checksum.txt
  3. Verify the md5sum of the compressed tarball:
    md5sum -c ossec-hids-2.8.1-checksum.txt
    The output should be: 
    			ossec-hids-2.8.1.tar.gz: OK
			md5sum: WARNING: 1 line is improperly formatted
  4. To verify SHA1 checksum use the following command:
    sha1sum -c ossec-hids-2.8.1-checksum.txt
    The output should be: 
    			ossec-hids-2.8.1.tar.gz: OK
			sha1sum: WARNING: 1 line is improperly formatted
    You can ignore the warnings, as the OK next tar output file confirms that the file is good to go.

Find your SMTP server

During installation, OSSEC will ask for your SMTP server. To determine the correct SMTP server follow these steps.

  1. You can use dig command to query your providers mail exchange resource record. Enter following command
    dig -t mx coolexample.com
    You will get answer in several sections but we are only interested in the ANSWER section. For example if you run the command using fastmail.com
    dig -t mx fastmail.com
    The valid SMTP servers for providers will be shown in the ANSWER section. 
    					;; ANSWER SECTION:
					fastmail.com.           3600    IN      MX      10 in1-smtp.messagingengine.com.
					fastmail.com.           3600    IN      MX      20 in2-smtp.messagingengine.com.
    So in this example you can use in1-smtp.messagingengine.com or in1-smtp.messagingengine.com as SMTP server.
  2. Make a note of any of these SMTP servers. We will need them during installation.

Installation

  1. You can install OSSEC in many mode including server, agent, local or hybrid mode. We will install in local mode in this article. Expand the file with
    sudo tar -zxf ossec-hids-2.8.1.tar.gz
    If you get an error for tar command type "sudo yum install tar".

    Now change the directory to the expanded one.

    cd ossec-hids-2.8.1
    Initiate the installation with:
    sudo ./install.sh
  2. It will prompt you to answer a few questions. First question will be to choose a language; you may choose your language as you want. To use default press Enter. 
    					OSSEC HIDS v2.8 Installation Script - http://www.ossec.net
					
					 You are about to start the installation process of the OSSEC HIDS.
					 You must have a C compiler pre-installed in your system.
					 If you have any questions or comments, please send an e-mail
					 to dcid@ossec.net (or daniel.cid@gmail.com).

					  - System: Linux ubuntu 3.13.0-55-generic
					  - User: root
					  - Host: ubuntu


					  -- Press ENTER to continue or Ctrl-C to abort. -
  3. When prompted to enter the type of installation, type local and press Enter.
    1- What kind of installation do you want (server, agent, local, hybrid or help)? local
  4. For the rest of the questions, you may press Enter to accept the defaults. However, in Question 3.1 it will prompt you for your email address and then SMTP server ip/host. Enter your email address and SMTP server that you got using the dig command.
  5. On successful installation you will get 
    				Thanks for using the OSSEC HIDS.
				If you have any question, suggestion or if you find any bug,
				contact us at contact@ossec.net or using our public maillist at
				ossec-list@ossec.net
				( http://www.ossec.net/main/support/ ).

				More information can be found at http://www.ossec.net

				---  Press ENTER to finish (maybe more information below). ---

    Press enter to finish the installation

Verify OSSEC's email settings

Here you will need to verify if the email credentials given by us and one that OSSEC auto-configured are correct.

  1. Email settings are in ossec.conf that resides in /var/ossec/etc. To access this file, switch to the root user

    sudo su

  2. Change the directoty to the configuration file's directory

    cd /var/ossec/etc

    You may want to make a backup copy of this file. Use the following command to do this.

    cp ossec.conf ossec.conf.00

  3. Open ossec.conf in any editor

  4. Email settings are at the top and following is the description of the fields

    • <email_to> is the email address given by you during installation
    • <email_from> is the address where the mails would appear to come from.
    • <smtp_server> is the SMTP server you gave during setup

  5. Update these settings as per your need and once you are finished, the final section will look like as shown.

    				<global>
					<email_notification>yes</email_notification>
					<email_to>user@coolexample.com</email_to>
					<smtp_server>mail.coolexample.com.</smtp_server>
					<email_from>user@coolexample.com</email_from>
				</global>

  6. Save the file and start the server using the following command:

    /var/ossec/bin/ossec-control start

    You should have received an email saying OSSEC has started.

Add Alerts

Now you will see how to change settings for OSSEC and its directory structure.

  1. Directory structure

    Its root directory (/var/ossec) can only be accessed with root privileges.

    • its main configuration file is in /var/ossec/etc
    • /var/ossec/rules contains predefined rules
    • /var/ossec/bin contains command used to manage OSSEC
    • /var/ossec/logs/ossec.log contains logs of the errors

  2. Main configuration file location

    The main configuration file is located at /var/ossec/etc/ossec.conf. First configuration option you will get is the email configuration you specified while installing. You can change if required at this place as explained in email verification steps

    			<global>
			<email_notification>yes</email_notification>
			<email_to>email@coolexample.com</email_to>
			<smtp_server>mail.coolexample.com.</smtp_server>
			<email_from>ossec@ossec_server</email_from>
			</global>

    To change the number of email per hour you can add here the following line

    <email_maxperhour>N</email_maxperhour>

    "N" is the number of emails. Make sure you <email_from> address contains a valid domain part as some third-party email provider may drop otherwise.

    If you are not getting emails as expected you can check logs at /var/ossec/logs/ossec.log

  3. Frequency of scan

    The <syscheck> section starts like this 
    				<syscheck>
				<!-- Frequency that syscheck is executed - default to every 22 hours -->
				<frequency>79200</frequency>

    So by default sytem check is run inevery 22 hours. You can change it. To enable alerts for new file creation, add the below mentioned line

    <alert_new_files>yes</alert_new_files>

    So it should read like:

    					<syscheck>
					<!-- Frequency that syscheck is executed - default to every 22 hours -->
					<frequency>79200</frequency>
					<alert_new_files>yes</alert_new_files>

  4. Directory and file configuration

    Right after that there is a list of system directory that it monitors as shown:

    				<!-- Directories to check  (perform all possible verifications) -->
				<directories check_all="yes">/etc,/usr/bin,/usr/sbin</directories>
				<directories check_all="yes">/bin,/sbin</directories>

    Add report_changes="yes" realtime="yes" to each line to enable real time monitoring.

    				<!-- Directories to check  (perform all possible verifications) -->
				<directories report_changes="yes" realtime="yes" check_all="yes">/etc,/usr/bin,/usr/sbin</directories>
				<directories report_changes="yes" realtime="yes" check_all="yes">/bin,/sbin</directories>

    You can add more directories to monitor here. To add /home/user_name and /var/www/ add the following line:

    <directories report_changes="yes" realtime="yes" restrict=".php|.js|.py|.sh|.html" check_all="yes">/home/user_name,/var/www</directories>

    Restrict setting tells only to monitor specified formats

  5. Local rules location

    Next files you would want to modify reside in /var/ossec/rules. In this directory if you use ls -lgG command, you will get a list of bunch of xml files but only two files local_rules.xml and ossec_rules.xml is of our use. The former file is used to add local rules and latter for default rules definitions. You will need sudo privileges to access these files.

    When a file is added to a monitored directory the rule 554 is fired from ossec_rules.xml. By default it doesn't send alert on firing of this rule. so you'll need to change the settings.

    1. Default rule definitions in ossec_rules.xml are useful to look at so we can modify and copy them into our local files. Rule 554 looks like:

      2. 					<rule id="554" level="0">
					<category>ossec</category>
					<decoded_as>syscheck_new_entry</decoded_as>
					<description>File added to the system.</description>
					<group>syscheck,</group>
					</rule>

      When level is set to "0" no alert will be sent. We will raise the alert level.

    2. Copy the rule 554 and open the file /var/ossec/rules/local_rules.xml in any editor and add the new rule at the end within the group tag. Change the level to "7"

      					<rule id="554" level="7" overwrite="yes">
					<category>ossec</category>
					<decoded_as>syscheck_new_entry</decoded_as>
					<description>File added to the system.</description>
					<group>syscheck,</group>
					</rule>
    3. Save and close the file. Now restart the OSSEC using:
      /var/ossec/bin/ossec-control restart

      4. You will get a mail when it is restarted.

  6. Triggering file change alerts

    When you add any file in directory to be monitored by OSSEC or you update or delete a file, you will receive the related alerts. It doesn't send real time notification for adding a file, except for modification or deletion. If you don't get emails, please check your spam folder or logs in /var/ossec/logs/ossec.log

Next Step

After following all these steps you will be able to setup OSSEC notifications. To look for more customization, you can refer to its Official Documentation.
