Install and configure Ghost with NGINX - Arch Linux

Difficulty: 2
Time: 1.0 hrs

Ghost is simple blogging platform, fast to setup and easy to use. Besides, it provides good customization options to the users. This article will assist you with the steps to set up a Ghost blogging platform with nginx on your Arch Linux platform.

Note: It is assumed that you are logged in as a root user.

Step 1: Prepare the machine

Login into the Fedora or CentOS machine and perform these steps using the commands described here. It is assumed that you have logged in as root user when performing the following tasks:
  1. Update your Fedora or CentOS server (for the interactive installations, you might need to respond regarding the confirmation for installation)
    pacman –Syu
  2. Install base development tools
    pacman –S base- devel

    When asked about the selection of member of base-devel, select all members by simply pressing Enter.

    pacman -S zip unzip unrar
    pacman -Sy git
    pacman -Sy nodejs
    pacman -Syu npm

Step 2: Install and Configure Ghost

  1. Download the latest version of Ghost from the site Ghost.org
    curl -L https://ghost.org/zip/ghost-latest.zip -o ghost.zip
  2. Unzip the Ghost to some suitable folder location (Recommended location is /var/www/ghost)
    mkdir –p /var/www
    unzip -uo ghost.zip -d /var/www/ghost
  3. Install the Ghost in the new ghost directory
    cd /var/www/ghost && npm install --production
  4. Start the Ghost (production environment) using the following command
    npm start --production
    The default port used by Ghost is 127.0.0.1:2368.

    When it starts, it should give a message “your blog is now available on http://<provided URL>. This value of URL can be updated in the config file config.js.
    To shut down the blog, and return to command prompt, press CTRL+C

Step 3: Install and Configure nginx

Nginx, an open-source, high performance HTTP Server, is used to enable connections from the outside on port 80 to connect through to the port that Ghost is running on, so that people can see your blog.
  1. Install nginx using the following command
    pacman -Syu nginx
  2. Open the ghost config file /var/www/ghost/config.js for editing
    vim /var/www/ghost/config.js
  3. Put the following code into the Ghost file (content highlighted in red needs to be updated). The URL could be your domain name (http://your-domainname.com) or the public IP address of your droplet 
      production: {
        url: 'http://192.169.165.197',
        mail: {},
        database: {
            client: 'sqlite3',
            connection: {
                filename: path.join(__dirname, '/content/data/ghost.db')
            },
            debug: false
        },

        server: {
            // Host to be passed to node's `net.Server#listen()`
            host: '192.169.165.197',
            // Port to be passed to node's `net.Server#listen()`, for iisnode set this to `process.env.PORT`
            port: '2368'
        }
    }
  4. Start the Ghost (production environment) using the following command
    cd /var/www/ghost
    npm start --production
  5. Now the Ghost blog should be available on the web browser via the specified url. In your browser, open the page (red highlighted text needs to be updated)

    http://192.169.164.127:2368/

Conclusion

The Ghost blogging platform, with its ease of deployment and use, can help individuals and organizations setup and extend their content management infrastructure. When used with nginx, and with availability of options for setting up and supporting multiple environments like production, development or testing, it can be easily adapted for any environment.
